# Contributing

Thanks for helping improve these guides for the next reader.

## Ways to Contribute

### 📝 Content
- **Fix errors** — typos, broken links, outdated commands, wrong formulas
- **Improve explanations** — rewrite confusing sections
- **Add examples** — especially runnable code
- **Add new articles** — expand existing series or propose a new topic

### 🌍 Translations
Create `translations/<lang-code>/` and mirror the structure. Partial translations welcome — one article at a time is fine.

### 🎓 Placement Prep Contributions
After your TCS NQT / Infosys / campus test, add questions you remember to the relevant prep folder (e.g., [TCS-Prep/06-previous-year-questions.md](TCS-Prep/06-previous-year-questions.md)). Include slot date + approach.

### 💻 Code
- Add solutions in other languages (Python, Java, C++, Go, Rust)
- Fix broken code samples
- Improve test coverage of examples

## Process

1. **Fork** the repo
2. **Create a branch**: `git checkout -b fix-kafka-part-3-typo`
3. **Make your changes** (one logical change per PR)
4. **Commit**: `git commit -m "Fix off-by-one in kafka-deep-dive-part-3 example"`
5. **Push & open a PR** — explain *what* and *why*

## Guidelines

- **Accuracy first.** Verify code compiles / runs. Verify formulas.
- **Keep the tone.** Code-first, from-scratch-to-production, no marketing fluff.
- **Markdown style:** GitHub-flavored. Use tables, code fences with language tags.
- **File naming:** lowercase-with-dashes for new files in existing folders; match existing pattern.
- **No paid promotions.** Free resources only.
- **No AI-generated slop** — if you use AI to draft, edit for accuracy & voice before submitting.

## Reporting Issues

Found a bug, error, or unclear section? [Open an issue](https://github.com/sachinchaurasiya360/My-Medium-Article/issues). Include:
- The file path and line (if applicable)
- What you expected vs what's there
- Optional: your suggested fix

## Code of Conduct

- Be respectful. Many readers are students under exam pressure.
- Assume good intent.
- No spam, no self-promotion, no off-topic.
- If you disagree with a solution, explain *why* with a reference.

## Recognition

Every contributor is listed via `git log`. Your PR helps thousands of future learners — that's your legacy here.

---

Questions about contributing? Open an issue or ping via the live site. Happy contributing.
